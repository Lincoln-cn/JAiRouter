### 2. 审计日志 Prometheus 指标

新增 `AuditMetricsService` 提供审计系统健康状态的监控指标：

**指标列表**:

| 指标名称 | 类型 | 说明 |
|----------|------|------|
| `jairouter_audit_events_total` | Counter | 审计事件总数 |
| `jairouter_audit_events_success_total` | Counter | 成功审计事件数 |
| `jairouter_audit_events_failure_total` | Counter | 失败审计事件数 |
| `jairouter_audit_fallback_events_total` | Counter | 备用存储事件数 |
| `jairouter_audit_primary_storage_failures_total` | Counter | 主存储失败次数 |
| `jairouter_audit_event_write_duration_seconds` | Timer | 写入耗时 |
| `jairouter_audit_health_score` | Gauge | 健康分数 (0-100) |
| `jairouter_audit_buffer_size` | Gauge | 缓冲区大小 |
| `jairouter_audit_total_events` | Gauge | 总事件数 |

**使用示例**:

```bash
# 查询审计事件总数
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_events_total

# 查询健康分数
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_health_score

# 查询写入耗时
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_event_write_duration
```

**Grafana 仪表盘示例**:

```promql
# 审计事件速率
rate(jairouter_audit_events_total[5m])

# 健康分数趋势
jairouter_audit_health_score

# 主存储失败率
rate(jairouter_audit_primary_storage_failures_total[5m]) / rate(jairouter_audit_events_total[5m])
```

## 🔧 配置变更

### 新增配置项

```yaml
# application.yml
jairouter:
  security:
    audit:
      # 启用审计指标导出（默认启用）
      metrics:
        enabled: true
```

### 环境变量

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| `JAiROUTER_SECURITY_AUDIT_METRICS_ENABLED` | 启用审计指标 | `true` |

## 📦 安装与升级

### 升级步骤

1. 备份当前配置和数据
2. 下载新版本 JAR
3. 重启应用

```bash
# 停止旧版本
docker stop jairouter

# 启动新版本
docker run -d \
  --name jairouter \
  -p 8080:8080 \
  -e JWT_SECRET="$JWT_SECRET" \
  -e INITIAL_ADMIN_PASSWORD="$INITIAL_ADMIN_PASSWORD" \
  sodlinken/jairouter:1.8.1
```

- ✅ 完全向后兼容 v1.8.0
- ✅ 配置格式无变化
- ✅ API 接口无变化
