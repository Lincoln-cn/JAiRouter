# JAiRouter v1.8.0 发布说明

> 版本：v1.8.0  
> 发布日期：2026 年 4 月 16 日  
> 版本主题：安全加固版本

---

## 📋 版本概述

v1.8.0 是 JAiRouter 的安全加固版本，主要聚焦于消除安全隐患、提升配置安全性和系统可靠性。

### 核心改进

- 🔐 **密钥安全管理** - 移除所有硬编码密钥，实现密钥强度检查
- 🔧 **配置验证机制** - 启动时自动验证关键配置合理性
- 📝 **审计日志增强** - 备用日志通道，确保审计数据不丢失

---

## 🚀 新特性

### 1. 密钥生成工具

新增命令行密钥生成工具，支持生成安全的随机密钥：

```bash
# 生成 JWT 密钥（Base64 编码）
java -jar jairouter.jar --generate-key

# 生成 API Token
java -jar jairouter.jar --generate-api-token

# 生成随机密码
java -jar jairouter.jar --generate-password
```

## 🔧 配置变更

### 环境变量要求

v1.8.0 开始，以下配置必须通过环境变量注入：

| 环境变量 | 说明 | 要求 |
|----------|------|------|
| `JWT_SECRET` | JWT 密钥 | 必须，至少 32 字节 |
| `INITIAL_ADMIN_PASSWORD` | 管理员初始密码 | 建议设置，至少 12 字符 |
| `GPUSTACK_API_TOKEN` | GPUStack API Token | 使用 GPUStack 服务时必须 |
| `DEV_JWT_SECRET` | 开发环境 JWT 密钥 | 开发环境可选 |
| `DEV_ADMIN_PASSWORD` | 开发环境管理员密码 | 开发环境可选 |

### 配置文件变更

**model-services-base.yml** - API Token 改为环境变量：
```yaml
# 之前（硬编码）
headers:
  Authorization: "Bearer gpustack_d9e4cb18169ca682_..."

# 之后（环境变量）
headers:
  Authorization: "Bearer ${GPUSTACK_API_TOKEN:your-api-token-here}"
```

**security-base.yml** - 管理员密码改为环境变量：
```yaml
# 之前（硬编码）
password: "{noop}UqfpTm2Zw7ff2BNnZb8AQo8t"

# 之后（环境变量）
password: "{noop}${INITIAL_ADMIN_PASSWORD:ChangeMeOnFirstStartup}"
```

## 📦 安装与升级

### Docker 部署

```bash
# 生成密钥
java -jar jairouter.jar --generate-key

# 设置环境变量
export JWT_SECRET="生成的密钥"
export INITIAL_ADMIN_PASSWORD="强密码"

# 启动容器
docker run -d \
  --name jairouter \
  -p 8080:8080 \
  -e JWT_SECRET="$JWT_SECRET" \
  -e INITIAL_ADMIN_PASSWORD="$INITIAL_ADMIN_PASSWORD" \
  -e GPUSTACK_API_TOKEN="your-api-token" \
  sodlinken/jairouter:latest
```

### Docker Compose

```yaml
version: '3.8'
services:
  jairouter:
    image: sodlinken/jairouter:latest
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - INITIAL_ADMIN_PASSWORD=${INITIAL_ADMIN_PASSWORD}
      - GPUSTACK_API_TOKEN=${GPUSTACK_API_TOKEN}
    ports:
      - "8080:8080"
```

### 升级步骤

1. 备份当前配置和数据
2. 生成新的密钥（如需要）
3. 更新环境变量配置
4. 重启应用
5. 验证启动日志中的安全检查结果

## ⚠️ 重要说明

### 安全警告

如果启动时检测到弱密钥或弱密码，应用会输出警告信息：

```
╔══════════════════════════════════════════════════════════════════════════════╗
║  ⚠️  安全警告：检测到弱密钥或弱密码                                          ║
║                                                                              ║
║  在生产环境使用之前，请务必：                                                ║
║  1. 设置强密钥：export JWT_SECRET="<随机生成的 32+ 字节密钥>"                    ║
║  2. 设置强密码：export INITIAL_ADMIN_PASSWORD="<复杂密码>"                     ║
║  3. 使用密钥生成工具：java -jar jairouter.jar --generate-key                 ║
╚══════════════════════════════════════════════════════════════════════════════╝
```

### 向后兼容性

- ✅ 配置格式兼容，无需修改现有配置文件
- ⚠️ 硬编码的密钥需要替换为环境变量引用
- ✅ 默认值仍然可用，但会发出安全警告
