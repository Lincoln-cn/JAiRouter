# 配置版本管理

> **版本：** 2.6.11  
> **最后更新：** 2026-06-09  
> **API 端点：** `/api/config/version`

## 概述

配置版本管理功能允许用户管理和恢复系统的不同配置状态。它提供全面的版本控制能力，包括版本列表、对比、应用和删除功能。

## REST API 端点

### 版本查询
- `GET /api/config/version` - 获取所有配置版本列表
- `GET /api/config/version/current` - 获取当前活动版本号
- `GET /api/config/version/{version}` - 获取指定版本配置详情
- `GET /api/config/version/info` - 获取所有版本的详细信息

### 版本操作
- `POST /api/config/version/apply/{version}` - 应用指定版本的配置
- `DELETE /api/config/version/{version}` - 删除指定版本（不能删除当前版本）

### 版本对比
- `GET /api/config/version/compare/{sourceVersion}/{targetVersion}` - 比较两个版本
- `GET /api/config/version/compare/{version}` - 查看特定版本的变更内容（与上一版本对比）

## 应用 (Apply) 操作

### 定义

**应用**操作是将指定历史版本的配置内容复制并设置为当前正在使用的配置。

### 特点

- **用途**：将某个历史版本的配置重新应用为当前工作配置
- **版本历史**：不改变现有的配置版本历史结构
- **语义**：更像是"使用这个配置"，强调的是配置内容的复用
- **审计**：在操作日志中记录为"应用配置"

### 使用场景

- 当您想重新使用之前某个特定配置的内容时
- 当您需要临时切换到历史配置进行测试时
- 当您确认某个历史配置是最佳配置并想将其设为当前配置时

## 版本对比

### 功能特性

- **并排对比**：比较任意两个版本，识别差异
- **变更追踪**：查看新增、删除和修改的内容
- **快速查看**：将版本与其前一个版本对比，查看变更内容

### 使用场景

- 理解配置的演变过程
- 审计配置变更
- 排查配置问题

## 最佳实践

### 何时使用应用操作

1. 当您想测试某个历史配置的效果时
2. 当您确认某个历史配置是最佳实践时
3. 当您需要临时切换配置进行对比时

### 何时使用版本对比

1. 应用历史版本前，先与当前配置对比
2. 了解两个版本之间的变化
3. 用于审计和合规性检查

## 操作步骤

### 应用配置版本

1. 在版本管理界面找到目标版本
2. 点击"应用"按钮
3. 确认操作提示
4. 等待系统应用配置
5. 界面自动刷新显示新的当前版本

### 对比版本

1. 选择源版本和目标版本
2. 点击"对比"按钮
3. 在对比视图中查看差异
4. 差异分类为：新增、修改、删除

### 删除版本

1. 选择要删除的版本（不能删除当前版本）
2. 点击"删除"按钮
3. 确认操作
4. 版本从历史记录中移除
