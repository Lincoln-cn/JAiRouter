## ✨ 新增功能

### 1. 熔断器状态模式重构 (P1-1)

**提交 ID**: `3e80254`

**重构内容**:
- 新增 `CircuitState` 状态接口
- 新增 `CircuitStateContext` 状态上下文
- 新增 `ClosedState` / `OpenState` / `HalfOpenState` 状态实现
- 新增 `StatefulCircuitBreaker` 状态模式熔断器

**代码改进**:
| 指标 | 重构前 | 重构后 | 改进 |
|------|--------|--------|------|
| 圈复杂度 | 15 | 5 | ↓ 67% |
| 代码行数 | 280 | 474 | +69% (结构更清晰) |
| 状态转换逻辑 | 分散在 switch 中 | 集中在状态类 | 易于维护 |

**设计优势**:
- ✅ 符合单一职责原则：每个状态类只负责一种状态的逻辑
- ✅ 符合开闭原则：添加新状态无需修改现有代码
- ✅ 降低维护成本：状态转换逻辑清晰，易于调试

**涉及文件**:
```
src/main/java/org/unreal/modelrouter/circuitbreaker/
├── StatefulCircuitBreaker.java          # 状态模式熔断器
└── state/
    ├── CircuitState.java                # 状态接口
    ├── CircuitStateContext.java         # 状态上下文
    ├── ClosedState.java                 # 关闭状态
    ├── OpenState.java                   # 打开状态
    └── HalfOpenState.java               # 半开状态
```

### 4. 值对象模式引入 (P2-7)

**提交 ID**: `1d663e8`

**新增组件**:
- `InstanceId` - 实例 ID 值对象

**设计优势**:
| 特性 | 说明 |
|------|------|
| 有效性验证 | 非空、长度限制 (≤64)、字符合法性 |
| 统一生成逻辑 | `generate()` 方法生成随机实例 ID |
| 类型安全 | 避免字符串误用，增强代码可读性 |
| 值语义 | 基于 value 的 equals/hashCode |

**使用示例**:
```java
// 从字符串创建（带验证）
InstanceId id = InstanceId.of("my-instance-001");

// 生成随机实例 ID
InstanceId newId = InstanceId.generate();

// 获取值
String value = id.getValue();  // 或 id.toString()
```

**涉及文件**:
```
src/main/java/org/unreal/modelrouter/vo/
└── InstanceId.java
```
