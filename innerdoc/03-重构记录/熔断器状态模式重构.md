# 熔断器状态模式重构

**重构日期**: 2026-03-24  
**版本**: V1.4.1  
**提交 ID**: `3e80254`

---

## 📋 重构背景

### 问题描述

原有的 `DefaultCircuitBreaker` 实现使用 `switch` 语句处理熔断器状态转换，存在以下问题：

1. **圈复杂度高** (15) - 多个状态判断和转换逻辑交织
2. **违反单一职责原则** - 一个类负责所有状态的逻辑
3. **难以扩展** - 添加新状态需要修改现有 switch 语句
4. **可维护性差** - 状态转换逻辑分散，难以追踪

### 重构目标

- 降低圈复杂度 (15 → 5)
- 符合单一职责原则
- 符合开闭原则
- 提高代码可维护性

## 🏗️ 设计方案

### 状态模式架构图

```
┌─────────────────────────────────────────────────────────┐
│                  CircuitStateContext                    │
│  - currentState: CircuitState                           │
│  - failureThreshold: int                                │
│  - successThreshold: int                                │
│  - timeout: long                                        │
├─────────────────────────────────────────────────────────┤
│  + recordSuccess()                                      │
│  + recordFailure()                                      │
│  + allowRequest(): boolean                              │
│  + transitionTo(CircuitState)                           │
└─────────────────────────────────────────────────────────┘
                            │
                            │ 委托
                            ▼
┌─────────────────────────────────────────────────────────┐
│                  <<interface>> CircuitState             │
├─────────────────────────────────────────────────────────┤
│  + getName(): String                                    │
│  + allowRequest(context): boolean                       │
│  + recordSuccess(context): CircuitState                 │
│  + recordFailure(context): CircuitState                 │
└─────────────────────────────────────────────────────────┘
           │                    │                    │
           │ 实现               │ 实现               │ 实现
           ▼                    ▼                    ▼
┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│   ClosedState    │  │    OpenState     │  │   HalfOpenState  │
│  (关闭状态)       │  │   (打开状态)      │  │   (半开状态)      │
├──────────────────┤  ├──────────────────┤  ├──────────────────┤
│ - 正常处理请求   │  │ - 拒绝所有请求   │  │ - 允许一个请求   │
│ - 失败计数       │  │ - 超时转换       │  │ - 成功→CLOSED    │
│ - 达到阈值→OPEN  │  │ - 超时→HALF_OPEN │  │ - 失败→OPEN      │
└──────────────────┘  └──────────────────┘  └──────────────────┘
```

### 类职责说明

| 类名 | 职责 |
|------|------|
| `CircuitState` | 状态接口，定义状态行为契约 |
| `CircuitStateContext` | 状态上下文，维护当前状态和熔断器配置 |
| `ClosedState` | 关闭状态实现，正常处理请求 |
| `OpenState` | 打开状态实现，拒绝所有请求 |
| `HalfOpenState` | 半开状态实现，试探性允许请求 |
| `StatefulCircuitBreaker` | 状态模式熔断器，对外提供统一接口 |

## 📝 实现细节

### 1. CircuitState 接口

```java
public interface CircuitState {
    /**
     * 获取状态名称
     */
    String getName();
    
    /**
     * 是否允许请求
     */
    boolean allowRequest(CircuitStateContext context);
    
    /**
     * 记录成功
     * @return 下一个状态
     */
    CircuitState recordSuccess(CircuitStateContext context);
    
    /**
     * 记录失败
     * @return 下一个状态
     */
    CircuitState recordFailure(CircuitStateContext context);
}
```

### 2. ClosedState 实现

```java
public class ClosedState implements CircuitState {
    @Override
    public String getName() {
        return "CLOSED";
    }
    
    @Override
    public boolean allowRequest(CircuitStateContext context) {
        return true;  // 关闭状态允许所有请求
    }
    
    @Override
    public CircuitState recordSuccess(CircuitStateContext context) {
        context.resetFailureCount();  // 重置失败计数
        return this;  // 保持关闭状态
    }
    
    @Override
    public CircuitState recordFailure(CircuitStateContext context) {
        context.incrementFailureCount();
        if (context.getFailureCount() >= context.getFailureThreshold()) {
            return CircuitStateContext.OPEN_STATE;  // 转换到打开状态
        }
        return this;  // 保持关闭状态
    }
}
```

### 3. OpenState 实现

```java
public class OpenState implements CircuitState {
    @Override
    public String getName() {
        return "OPEN";
    }
    
    @Override
    public boolean allowRequest(CircuitStateContext context) {
        // 检查是否超过超时时间
        long elapsed = System.currentTimeMillis() - context.getLastFailureTime();
        if (elapsed >= context.getTimeout()) {
            return true;  // 超时后允许请求（实际会转换到 HALF_OPEN）
        }
        return false;  // 拒绝所有请求
    }
    
    @Override
    public CircuitState recordSuccess(CircuitStateContext context) {
        return CircuitStateContext.HALF_OPEN_STATE;  // 超时后转换到半开状态
    }
    
    @Override
    public CircuitState recordFailure(CircuitStateContext context) {
        context.setLastFailureTime(System.currentTimeMillis());
        return this;  // 保持打开状态
    }
}
```

### 4. HalfOpenState 实现

```java
public class HalfOpenState implements CircuitState {
    @Override
    public String getName() {
        return "HALF_OPEN";
    }
    
    @Override
    public boolean allowRequest(CircuitStateContext context) {
        return true;  // 允许一个请求试探
    }
    
    @Override
    public CircuitState recordSuccess(CircuitStateContext context) {
        context.resetFailureCount();
        return CircuitStateContext.CLOSED_STATE;  // 成功则回到关闭状态
    }
    
    @Override
    public CircuitState recordFailure(CircuitStateContext context) {
        context.setLastFailureTime(System.currentTimeMillis());
        return CircuitStateContext.OPEN_STATE;  // 失败则回到打开状态
    }
}
```

### 可维护性提升

| 方面 | 改进说明 |
|------|----------|
| **可读性** | 每个状态类职责单一，代码更易理解 |

## 🧪 测试用例

### 状态转换测试

```java
@Test
public void testClosedToOpenTransition() {
    StatefulCircuitBreaker breaker = new StatefulCircuitBreaker(3, 2, 5000);
    
    // 记录 3 次失败，应该转换到 OPEN 状态
    breaker.recordFailure();
    breaker.recordFailure();
    breaker.recordFailure();
    
    assertEquals("OPEN", breaker.getState());
}

@Test
public void testOpenToHalfOpenTransition() throws InterruptedException {
    StatefulCircuitBreaker breaker = new StatefulCircuitBreaker(3, 2, 100);
    
    // 先转换到 OPEN 状态
    for (int i = 0; i < 3; i++) {
        breaker.recordFailure();
    }
    
    // 等待超时
    Thread.sleep(150);
    
    // 应该允许请求（转换到 HALF_OPEN）
    assertTrue(breaker.allowRequest());
}

@Test
public void testHalfOpenToClosedTransition() {
    StatefulCircuitBreaker breaker = new StatefulCircuitBreaker(3, 2, 100);
    
    // 进入 OPEN 状态
    for (int i = 0; i < 3; i++) {
        breaker.recordFailure();
    }
    
    // 等待超时，进入 HALF_OPEN
    try { Thread.sleep(150); } catch (InterruptedException e) {}
    breaker.allowRequest();
    
    // 记录成功，应该回到 CLOSED
    breaker.recordSuccess();
    assertEquals("CLOSED", breaker.getState());
}
```

## ⚠️ 注意事项

### 向后兼容性

- ✅ 保留 `DefaultCircuitBreaker` 类
- ✅ 新增 `StatefulCircuitBreaker` 不破坏现有代码
- ✅ 配置参数保持一致

### 使用建议

```java
// 推荐使用新的状态模式熔断器
CircuitBreaker breaker = new StatefulCircuitBreaker(
    5,      // 失败阈值
    2,      // 成功阈值
    60000   // 超时时间 (ms)
);

// 记录请求结果
if (requestSuccess) {
    breaker.recordSuccess();
} else {
    breaker.recordFailure();
}

// 检查是否允许请求
if (breaker.allowRequest()) {
    // 执行请求
} else {
    // 快速失败
}
```
