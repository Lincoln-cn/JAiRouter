## 文档维护说明

### 文档更新流程

1. **创建文档**：在对应目录下创建 Markdown 文件
2. **更新索引**：在本文档中添加或更新链接
3. **审查文档**：确保文档内容准确、完整
4. **版本控制**：重要文档变更需要提交 Git

### 文档命名规范

- 使用中文文件名
- 使用短横线 `-` 分隔词语
- 包含日期时在文件名末尾添加 `-YYYYMMDD`

### 文档模板

```markdown
# 文档标题

> 描述信息  
> 最后更新：YYYY-MM-DD

---

## 章节 1

内容...

## 章节 2

内容...

---

**文档结束**
```
