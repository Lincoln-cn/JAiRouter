# 前端代理配置修复说明

**日期**: 2026-03-13  
**问题**: 前端代理地址配置错误（8080 → 8081）  
**状态**: ✅ 已修复

---

## 问题描述

服务启动在 **8081** 端口，但前端代理配置的是 **8080** 端口，导致前端页面登录失败。

---

## 修复内容

### 1. 更新 Vite 配置文件

**文件**: `frontend/vite.config.ts`

**修改前**:
```typescript
server: {
  proxy: {
    '/api': {
      target: process.env.VITE_API_URL || 'http://127.0.0.1:8080',
      // ...
    }
  }
}
```

**修改后**:
```typescript
server: {
  proxy: {
    '/api': {
      target: process.env.VITE_API_URL || 'http://127.0.0.1:8081',
      // ...
    }
  }
}
```

### 2. 更新环境变量文件

**文件**: `frontend/.env` 和 `frontend/.env.production`

**修改后**:
```bash
# Development environment variables
VITE_API_BASE_URL=/api
VITE_API_URL=http://127.0.0.1:8081
VITE_APP_TITLE=JAiRouter Admin
```

## 代理配置说明

### Vite 代理工作原理

```
浏览器请求 → Vite 开发服务器 (3000) → 代理转发 → 后端服务 (8081)
```

### 代理配置

```typescript
proxy: {
  '/api': {
    target: 'http://127.0.0.1:8081',  // 后端地址
    changeOrigin: true,                // 修改 Host 头
    secure: false,                     // 允许 HTTPS
  }
}
```

### 请求示例

**前端代码**:
```javascript
axios.get('/api/v1/models')
```

**实际请求**:
```
GET http://localhost:8081/v1/models
```
