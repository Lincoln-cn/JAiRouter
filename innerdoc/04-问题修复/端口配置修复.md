# 端口配置修复说明

**日期**: 2026-03-13  
**问题**: 服务启动在 8081 端口，但前端代理配置是 8080  
**解决方案**: 恢复后端到 8080 端口

---

## 修复内容

### 1. 恢复后端端口到 8080

**启动命令**:
```bash
java -jar target/model-router-1.2.5.jar --server.port=8080
```

### 2. 更新前端代理配置回 8080

**文件**: `frontend/vite.config.ts`
```typescript
proxy: {
  '/api': {
    target: process.env.VITE_API_URL || 'http://127.0.0.1:8080',
    // ...
  }
}
```

**文件**: `frontend/.env` 和 `frontend/.env.production`
```bash
VITE_API_URL=http://127.0.0.1:8080
```

### 3. 修复 WebAdmin 静态资源配置

**文件**: `src/main/java/org/unreal/modelrouter/config/WebAdminResourceConfig.java`

修复了 SPA 路径解析器，确保前端路由正确工作。

## 快速启动

### 使用启动脚本
```bash
bash scripts/dev-start.sh
```

### 手动启动
```bash
# 启动后端
java -jar target/model-router-1.2.5.jar \
  --server.port=8080 \
  --spring.profiles.active=dev \
  --store.type=h2

# 启动前端（开发模式）
cd frontend && npm run dev
```
