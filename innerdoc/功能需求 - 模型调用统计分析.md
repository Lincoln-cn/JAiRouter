## 一、功能概述

### 1.1 业务背景

随着 JAiRouter 路由的 AI 模型服务增多，用户需要了解：
- 各个模型的使用频率和分布
- 模型调用的性能和成本
- Token 消耗情况和趋势
- 服务质量和响应速度

### 1.2 功能目标

在 Admin 控制台「概览」菜单下，新增**模型调用统计**子菜单，提供：
- 模型调用量统计
- Token 使用情况分析
- 响应性能指标（TTFT、总耗时）
- 错误率和成功率分析
- 实时调用监控

## 二、统计维度设计

### 2.1 核心指标体系

```
模型调用统计
├── 调用量统计
│   ├── 总调用次数
│   ├── 成功/失败次数
│   ├── 按模型分组
│   ├── 按服务类型分组 (Chat/Embedding/Rerank/TTS/STT)
│   └── 时间趋势 (小时/天/周/月)
│
├── Token 统计
│   ├── 总 Token 消耗量
│   ├── 输入 Token (prompt_tokens)
│   ├── 输出 Token (completion_tokens)
│   ├── Token 成本估算
│   ├── 按模型分组统计
│   └── Token 使用趋势
│
├── 性能指标
│   ├── 平均响应时间
│   ├── P50/P90/P95/P99 延迟
│   ├── 首 Token 时间 (TTFT)
│   ├── 生成速度 (tokens/s)
│   └── 性能趋势分析
│
├── 质量指标
│   ├── 成功率
│   ├── 错误率
│   ├── 熔断触发次数
│   ├── 限流拒绝次数
│   └── 重试成功率
│
└── 成本分析
    ├── 按模型统计成本
    ├── 按服务类型统计
    ├── 日均/月均成本
    └── 成本趋势预测
```

### 2.2 详细指标定义

#### 调用量指标

| 指标名称 | 英文标识 | 类型 | 说明 |
|---------|---------|------|------|
| 总调用次数 | `total_calls` | Counter | 所有模型调用总次数 |
| 成功调用次数 | `successful_calls` | Counter | 返回 2xx 的调用次数 |
| 失败调用次数 | `failed_calls` | Counter | 返回 4xx/5xx 的调用次数 |
| 模型调用分布 | `calls_by_model` | Gauge | 按模型分组的调用次数 |
| 服务类型分布 | `calls_by_service` | Gauge | Chat/Embedding 等分布 |

#### Token 指标

| 指标名称 | 英文标识 | 类型 | 说明 |
|---------|---------|------|------|
| 总 Token 数 | `total_tokens` | Counter | 输入 + 输出 Token 总和 |
| 输入 Token 数 | `prompt_tokens` | Counter | 所有请求的输入 Token 总和 |
| 输出 Token 数 | `completion_tokens` | Counter | 所有响应的输出 Token 总和 |
| Token 成本 | `token_cost_usd` | Gauge | 根据模型单价估算的成本 |
| 平均 Token/请求 | `avg_tokens_per_call` | Gauge | 平均每次调用的 Token 数 |

#### 性能指标

| 指标名称 | 英文标识 | 类型 | 说明 |
|---------|---------|------|------|
| 平均响应时间 | `avg_response_time_ms` | Gauge | 所有请求的平均响应时间 |
| P50 延迟 | `p50_latency_ms` | Gauge | 50% 请求的响应时间 |
| P90 延迟 | `p90_latency_ms` | Gauge | 90% 请求的响应时间 |
| P95 延迟 | `p95_latency_ms` | Gauge | 95% 请求的响应时间 |
| P99 延迟 | `p99_latency_ms` | Gauge | 99% 请求的响应时间 |
| 首 Token 时间 | `ttft_ms` | Gauge | Time To First Token |
| 生成速度 | `generation_speed` | Gauge | tokens/second |

#### 质量指标

| 指标名称 | 英文标识 | 类型 | 说明 |
|---------|---------|------|------|
| 成功率 | `success_rate` | Gauge | 成功调用占比 (0-100%) |
| 错误率 | `error_rate` | Gauge | 失败调用占比 (0-100%) |
| 熔断次数 | `circuit_breaker_trips` | Counter | 熔断器打开次数 |
| 限流次数 | `rate_limit_rejections` | Counter | 被限流拒绝的次数 |
| 重试成功数 | `retry_successes` | Counter | 重试后成功的次数 |

## 三、数据存储设计

### 3.1 指标存储表

```sql
-- 模型调用统计表
CREATE TABLE model_call_stats (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    model_name VARCHAR(255) NOT NULL,
    service_type VARCHAR(50) NOT NULL,     -- chat, embedding, rerank, tts, stt
    adapter_type VARCHAR(50) NOT NULL,     -- gpustack, ollama, vllm, etc.
    instance_name VARCHAR(255) NOT NULL,   -- 实例名称
    
    call_count BIGINT DEFAULT 0,           -- 调用次数
    success_count BIGINT DEFAULT 0,        -- 成功次数
    failure_count BIGINT DEFAULT 0,        -- 失败次数
    
    total_tokens BIGINT DEFAULT 0,         -- 总 Token 数
    prompt_tokens BIGINT DEFAULT 0,        -- 输入 Token 数
    completion_tokens BIGINT DEFAULT 0,    -- 输出 Token 数
    
    total_response_time_ms BIGINT DEFAULT 0,  -- 总响应时间
    ttft_total_ms BIGINT DEFAULT 0,           -- TTFT 总和
    ttft_count BIGINT DEFAULT 0,              -- TTFT 记录次数
    
    time_window TIMESTAMP NOT NULL,        -- 时间窗口 (小时)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_model_time (model_name, time_window),
    INDEX idx_service_time (service_type, time_window),
    INDEX idx_time (time_window)
);

-- Token 使用明细表 (用于详细分析)
CREATE TABLE token_usage_details (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    trace_id VARCHAR(64) NOT NULL,
    model_name VARCHAR(255) NOT NULL,
    service_type VARCHAR(50) NOT NULL,
    
    prompt_tokens INT NOT NULL,
    completion_tokens INT NOT NULL,
    total_tokens INT NOT NULL,
    
    response_time_ms INT NOT NULL,
    ttft_ms INT,
    
    status VARCHAR(20) NOT NULL,           -- success, failure
    error_code VARCHAR(50),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_trace (trace_id),
    INDEX idx_model_time (model_name, created_at),
    INDEX idx_time (created_at)
);

-- 性能指标汇总表 (按小时聚合)
CREATE TABLE performance_metrics_hourly (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    model_name VARCHAR(255) NOT NULL,
    service_type VARCHAR(50) NOT NULL,
    hour TIMESTAMP NOT NULL,
    
    call_count INT NOT NULL,
    avg_response_time_ms DECIMAL(10,2),
    p50_response_time_ms INT,
    p90_response_time_ms INT,
    p95_response_time_ms INT,
    p99_response_time_ms INT,
    
    avg_ttft_ms DECIMAL(10,2),
    avg_generation_speed DECIMAL(10,2),
    
    success_rate DECIMAL(5,2),
    error_rate DECIMAL(5,2),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_model_hour (model_name, hour),
    INDEX idx_time (hour)
);
```

### 3.2 数据保留策略

| 数据类型 | 保留期限 | 聚合粒度 |
|---------|---------|---------|
| 原始调用记录 | 7 天 | 每次调用 |
| 小时级聚合 | 30 天 | 每小时 |
| 天级聚合 | 365 天 | 每天 |
| 月度汇总 | 永久 | 每月 |

## 六、技术实现方案

### 6.1 后端实现

```java
// 统计服务接口
public interface ModelStatsService {
    
    // 记录调用统计
    void recordCall(ModelCallContext context);
    
    // 查询概览
    Mono<ModelStatsOverview> getOverview(TimeRange range);
    
    // 查询详细统计
    Mono<ModelDetailStats> getDetailStats(String model, TimeRange range);
    
    // 查询 Token 使用
    Mono<TokenUsageStats> getTokenUsage(TimeRange range);
    
    // 查询性能指标
    Mono<PerformanceStats> getPerformanceStats(TimeRange range);
    
    // 实时统计 (SSE)
    Flux<RealtimeCallStats> getRealtimeStats();
}

// 统计记录器
@Component
public class ModelStatsRecorder {
    
    private final ModelStatsRepository repository;
    private final MetricsCollector metricsCollector;
    
    // 在 BaseAdapter 中调用
    public void recordAfterCall(CallResult result) {
        // 提取 Token 信息
        int promptTokens = extractPromptTokens(result);
        int completionTokens = extractCompletionTokens(result);
        
        // 提取性能指标
        long ttft = extractTTFT(result);
        long responseTime = result.getDuration();
        
        // 保存到数据库
        repository.save(ModelCallStats.builder()
            .modelName(result.getModel())
            .serviceType(result.getServiceType())
            .callCount(1)
            .promptTokens(promptTokens)
            .completionTokens(completionTokens)
            .totalResponseTimeMs(responseTime)
            .ttftTotalMs(ttft)
            .ttftCount(1)
            .timeWindow(truncateToHour(Instant.now()))
            .build());
    }
}
```

```java
// Prometheus 指标注册
@Configuration
public class MetricsConfig {
    
    @Bean
    public MeterBinder modelStatsBinder(ModelStatsRepository repository) {
        return (registry) -> {
            // 注册模型调用指标
            Gauge.builder("jairouter.model.calls.total", repository, 
                    ModelStatsRepository::countTotalCalls)
                .description("Total model calls")
                .register(registry);
            
            // 注册 Token 指标
            Gauge.builder("jairouter.model.tokens.total", repository,
                    ModelStatsRepository::countTotalTokens)
                .description("Total tokens consumed")
                .register(registry);
            
            // 注册延迟指标
            Gauge.builder("jairouter.model.latency.avg", repository,
                    ModelStatsRepository::getAverageLatency)
                .description("Average model latency")
                .baseUnit("milliseconds")
                .register(registry);
        };
    }
}
```

### 6.3 定时聚合任务

```java
@Component
public class StatsAggregationTask {
    
    // 每小时聚合
    @Scheduled(cron = "0 5 * * * *")
    public void aggregateHourlyStats() {
        // 聚合上一小时的数据
        LocalDateTime lastHour = LocalDateTime.now().minusHours(1);
        repository.aggregateToHourly(lastHour);
    }
    
    // 每天聚合
    @Scheduled(cron = "0 10 0 * * *")
    public void aggregateDailyStats() {
        LocalDate yesterday = LocalDate.now().minusDays(1);
        repository.aggregateToDaily(yesterday);
    }
}
```

| 风险 | 影响 | 缓解措施 |

| 影响调用性能 | 中 | 异步记录 + 批量写入 |
| Token 解析失败 | 低 | 容错处理 + 默认值 |
