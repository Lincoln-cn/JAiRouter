### 2. Audit Log Prometheus Metrics

New `AuditMetricsService` provides monitoring metrics for audit system health:

**Metrics List**:

| Metric Name | Type | Description |
|-------------|------|-------------|
| `jairouter_audit_events_total` | Counter | Total audit events |
| `jairouter_audit_events_success_total` | Counter | Successful audit events |
| `jairouter_audit_events_failure_total` | Counter | Failed audit events |
| `jairouter_audit_fallback_events_total` | Counter | Fallback storage events |
| `jairouter_audit_primary_storage_failures_total` | Counter | Primary storage failures |
| `jairouter_audit_event_write_duration_seconds` | Timer | Write duration |
| `jairouter_audit_health_score` | Gauge | Health score (0-100) |
| `jairouter_audit_buffer_size` | Gauge | Buffer size |
| `jairouter_audit_total_events` | Gauge | Total events |

**Usage Example**:

```bash
# Query total audit events
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_events_total

# Query health score
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_health_score

# Query write duration
curl http://localhost:8080/actuator/prometheus | grep jairouter_audit_event_write_duration
```

**Grafana Dashboard Examples**:

```promql
# Audit event rate
rate(jairouter_audit_events_total[5m])

# Health score trend
jairouter_audit_health_score

# Primary storage failure rate
rate(jairouter_audit_primary_storage_failures_total[5m]) / rate(jairouter_audit_events_total[5m])
```

## 📦 Installation & Upgrade

### Upgrade Steps

1. Backup current configuration and data
2. Download new version JAR
3. Restart application

```bash
# Stop old version
docker stop jairouter

# Start new version
docker run -d \
  --name jairouter \
  -p 8080:8080 \
  -e JWT_SECRET="$JWT_SECRET" \
  -e INITIAL_ADMIN_PASSWORD="$INITIAL_ADMIN_PASSWORD" \
  sodlinken/jairouter:1.8.1
```

### Compatibility

- ✅ Fully backward compatible with v1.8.0
- ✅ No configuration format changes
- ✅ No API changes
