## 四、架构演进规划

### 4.1 短期架构改进（3 个月内）

```
当前架构：
┌─────────────────────────────────────────┐
│  WebFlux + Spring Security              │
│  ├─ Adapter Layer (monolithic)          │
│  ├─ CircuitBreaker (synchronized)       │
│  ├─ RateLimiter (ConcurrentHashMap)     │
│  └─ LoadBalancer (basic strategies)     │
└─────────────────────────────────────────┘

改进后架构：
┌─────────────────────────────────────────┐
│  WebFlux + Spring Security              │
│  ├─ Adapter Layer (modular)             │
│  │   ├─ RequestTransformTemplate        │
│  │   ├─ ResponseTransformer             │
│  │   └─ StreamingHandler                │
│  ├─ CircuitBreaker (lock-free)          │
│  ├─ RateLimiter (Redis-backed)          │
│  └─ LoadBalancer (adaptive)             │
└─────────────────────────────────────────┘
```

## 六、性能优化方案

### 6.1 JVM 优化配置

```bash
# 当前配置
-Xms512m -Xmx1024m -XX:+UseG1GC

# 生产环境建议优化
-Xms2g -Xmx2g                            # 固定堆大小，避免动态调整
-XX:MaxGCPauseMillis=100                 # 控制 GC 停顿时间
-XX:G1HeapRegionSize=16m                 # G1 区域大小
-XX:InitiatingHeapOccupancyPercent=45    # 触发并发 GC 阈值
-XX:+ParallelRefProcEnabled              # 并行处理引用
-XX:+UseStringDeduplication              # 字符串去重
```

### 6.2 响应式优化

```java
// 当前实现（潜在阻塞）
Mono.fromCallable(() -> {
    // 同步数据库操作
    return repository.findById(id);
});

// 优化实现（使用弹性调度器）
Mono.fromCallable(() -> {
    // 同步数据库操作
    return repository.findById(id);
}).subscribeOn(Schedulers.boundedElastic());

// 或使用响应式数据源
repository.findById(id); // 直接返回 Mono
```

### 6.3 缓存策略优化

| 数据类型 | 当前策略 | 建议优化 | 预期收益 |
|----------|----------|----------|----------|
| API Key | 内存缓存 | Redis 缓存 + 本地 Caffeine 二级缓存 | 降低 Redis 访问延迟 |
| JWT Token | 内存 + 文件 | Redis 集群 + 过期预清理 | 提升吊销效率 |
| 配置数据 | 内存缓存 | 配置变更时失效 + 定期刷新 | 保证配置一致性 |
| 限流计数 | 内存 | Redis + Lua 脚本（分布式限流） | 支持多实例限流 |

### 7.2 告警规则建议

```yaml
groups:
  - name: jairouter_alerts
    rules:
      # 适配器延迟告警
      - alert: HighAdapterLatency
        expr: histogram_quantile(0.95, adapter_request_duration_seconds_bucket) > 2
        for: 5m
        annotations:
          summary: "适配器 P95 延迟超过 2 秒"
          severity: warning

      # 熔断器打开告警
      - alert: CircuitBreakerOpen
        expr: circuit_breaker_state == 1
        for: 1m
        annotations:
          summary: "熔断器处于打开状态"
          severity: critical

      # 限流阈值告警
      - alert: RateLimitThreshold
        expr: rate_limit_remaining / rate_limit_max < 0.1
        for: 5m
        annotations:
          summary: "限流余量不足 10%"
          severity: warning

      # 错误率告警
      - alert: HighErrorRate
        expr: rate(http_server_requests_seconds_count{status=~"5.."}) / rate(http_server_requests_seconds_count) > 0.05
        for: 5m
        annotations:
          summary: "错误率超过 5%"
          severity: critical

      # 审计日志失败告警
      - alert: AuditLogFailures
        expr: rate(audit_log_write_failures_total[5m]) > 0
        for: 5m
        annotations:
          summary: "审计日志写入失败"
          severity: warning
```

        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'
          cache: maven
      - name: Run Quality Checks
        run: mvn checkstyle:check spotbugs:check
      - name: Run Tests

      - name: Coverage Check
        run: mvn jacoco:check
      - name: Security Scan

  build:
    needs: quality-gate

    steps:
      - uses: actions/checkout@v3
      - name: Build with Maven

      - name: Build Docker Image

  deploy-dev:

    if: github.ref == 'refs/heads/develop'
