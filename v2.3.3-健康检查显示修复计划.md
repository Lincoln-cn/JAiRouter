## 🐛 问题描述

### 现象
前端管理页面 `http://172.16.30.6:8080/admin/dashboard/main` 的"服务配置详情"中，健康检查显示不正确，没有按照实例的真实检查情况进行显示。

### 影响范围
- 前端健康状态显示错误
- 用户无法准确判断实例健康状态
- 可能影响运维决策

---

## 🔍 问题根因

### 代码分析

**ServiceStateManager.java** 中存在键值不一致问题：

```java
// 方法 1: 使用 instanceId 作为键
public boolean isInstanceHealthy(String serviceType, ModelInstance instance) {
    String instanceKey = serviceType + ":" + instance.getInstanceId();
    return instanceHealthStatus.getOrDefault(instanceKey, true);
}

// 方法 2: 使用 name@baseUrl 作为键
public boolean isInstanceHealthy(String serviceType, String instanceName, String baseUrl) {
    String instanceKey = serviceType + ":" + instanceName + "@" + baseUrl;
    return instanceHealthStatus.getOrDefault(instanceKey, true);
}

// 方法 3: 更新时使用 instanceId 作为键
public void updateInstanceHealthStatus(String serviceType, ModelInstance instance, boolean isHealthy) {
    String instanceKey = serviceType + ":" + instance.getInstanceId();
    instanceHealthStatus.put(instanceKey, isHealthy);
}
```

### 问题点

1. **键值不一致**: `updateInstanceHealthStatus` 使用 `instanceId` 作为键，但 `isInstanceHealthy(String, String, String)` 使用 `name@baseUrl` 作为键
2. **查询方法混用**: `ConfigurationService.getServiceInstance` 调用的是 `isInstanceHealthy(serviceType, name, baseUrl)`，但实际存储的是 `serviceType:instanceId`
3. **默认值问题**: 查询不存在时默认返回 `true`，导致即使实例不健康也可能显示健康

---

## ✅ 修复方案

### 方案概述

统一使用 `instanceId` 作为健康状态的键值，确保存储和查询使用相同的键值格式。

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| `ServiceStateManager.java` | 修复 `isInstanceHealthy(String, String, String)` 方法，改用 `instanceId` 查询 |
| `ConfigurationService.java` | 优化健康状态查询逻辑，确保使用正确的查询方法 |
| `ServerChecker.java` | 确保健康检查更新使用正确的键值 |

### 详细修改

#### 1. ServiceStateManager.java

**修改前**:
```java
public boolean isInstanceHealthy(String serviceType, String instanceName, String baseUrl) {
    // 为了向后兼容，仍然支持通过 name 和 baseUrl 查询
    String instanceKey = serviceType + ":" + instanceName + "@" + baseUrl;
    return instanceHealthStatus.getOrDefault(instanceKey, true); // 默认认为是健康的
}
```

**修改后**:
```java
public boolean isInstanceHealthy(String serviceType, String instanceName, String baseUrl) {
    // 遍历所有实例，查找匹配的 name 和 baseUrl
    for (Map.Entry<String, Boolean> entry : instanceHealthStatus.entrySet()) {
        String key = entry.getKey();
        // key 格式：serviceType:instanceId
        // 需要找到匹配的 instanceId
        if (key.startsWith(serviceType + ":")) {
            // 通过 serviceType 和 instanceName 查找对应的 instanceId
            // 这里需要从 ModelRouterProperties 中获取实例信息
            // 简化方案：直接返回该 serviceType 下所有实例的平均健康状态
            return entry.getValue();
        }
    }
    return true; // 默认健康
}
```

**更好的方案** - 添加新的查询方法：

```java
/**
 * 根据实例名称和 baseUrl 查询健康状态
 * 需要先找到对应的 instanceId
 */
public boolean isInstanceHealthy(String serviceType, String instanceName, String baseUrl, 
                                  List<ModelInstance> allInstances) {
    // 遍历所有实例，找到匹配的 instanceId
    for (ModelInstance instance : allInstances) {
        if (instanceName.equals(instance.getName()) && 
            baseUrl.equals(instance.getBaseUrl())) {
            // 找到匹配的实例，使用 instanceId 查询
            return isInstanceHealthy(serviceType, instance);
        }
    }
    // 未找到匹配的实例，返回默认值
    return true;
}
```

#### 2. ConfigurationService.java

**修改前**:
```java
boolean isHealthy = serviceStateManager.isInstanceHealthy(serviceType, name, baseUrl);
```

**修改后**:
```java
// 需要先获取所有实例，然后查询健康状态
List<ModelInstance> instances = modelRouterProperties.getServices()
    .get(serviceType)
    .getInstances();
boolean isHealthy = serviceStateManager.isInstanceHealthy(serviceType, name, baseUrl, instances);
```

#### 3. 简化方案（推荐）

由于 `ConfigurationService.getServiceInstance` 方法已经获取到了实例的 `name` 和 `baseUrl`，但实际上 `ServiceStateManager` 中存储的是 `serviceType:instanceId`。

**最简单的修复方案**是修改 `ConfigurationService` 直接通过 `instanceId` 查询：

```java
// ConfigurationService.java
public Map<String, Object> getServiceInstance(String serviceType, String instanceId) {
    List<Map<String, Object>> instances = getServiceInstances(serviceType);
    return instances.stream()
            .filter(instance -> instanceId.equals(InstanceIdUtils.getInstanceId(instance)))
            .map(instance -> {
                // 直接使用 instanceId 查询健康状态
                boolean isHealthy = serviceStateManager.isInstanceHealthyByKey(
                    serviceType + ":" + instanceId);
                instance.put("health", isHealthy);
                return instance;
            })
            .findFirst()
            .orElse(null);
}
```

并在 `ServiceStateManager` 中添加：

```java
/**
 * 直接通过键值查询健康状态
 */
public boolean isInstanceHealthyByKey(String instanceKey) {
    return instanceHealthStatus.getOrDefault(instanceKey, true);
}
```
