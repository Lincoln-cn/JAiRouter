# JAiRouter v1.8.0 Release Notes

> Version: v1.8.0  
> Release Date: April 16, 2026  
> Theme: Security Hardening Release

---

## 📋 Overview

v1.8.0 is a security hardening release for JAiRouter, focusing on eliminating security vulnerabilities, improving configuration security, and enhancing system reliability.

### Key Improvements

- 🔐 **Secret Key Management** - Remove all hardcoded keys, implement key strength validation
- 🔧 **Configuration Validation** - Automatically validate critical configurations on startup
- 📝 **Enhanced Audit Logging** - Fallback log channel to ensure audit data is not lost

---

## 🚀 New Features

## 🔧 Configuration Changes

### Environment Variables Requirements

Starting from v1.8.0, the following configurations must be injected via environment variables:

| Environment Variable | Description | Requirement |
|---------------------|-------------|-------------|
| `JWT_SECRET` | JWT secret key | Required, at least 32 bytes |
| `INITIAL_ADMIN_PASSWORD` | Admin initial password | Recommended, at least 12 characters |
| `GPUSTACK_API_TOKEN` | GPUStack API Token | Required when using GPUStack |
| `DEV_JWT_SECRET` | Development JWT secret | Optional for dev environment |
| `DEV_ADMIN_PASSWORD` | Development admin password | Optional for dev environment |

### Configuration File Changes

**model-services-base.yml** - API Token changed to environment variable:
```yaml
# Before (hardcoded)
headers:
  Authorization: "Bearer gpustack_d9e4cb18169ca682_..."

# After (environment variable)
headers:
  Authorization: "Bearer ${GPUSTACK_API_TOKEN:your-api-token-here}"
```

**security-base.yml** - Admin password changed to environment variable:
```yaml
# Before (hardcoded)
password: "{noop}UqfpTm2Zw7ff2BNnZb8AQo8t"

# After (environment variable)
password: "{noop}${INITIAL_ADMIN_PASSWORD:ChangeMeOnFirstStartup}"
```

## 📦 Installation & Upgrade

### Docker Deployment

```bash
# Generate key
java -jar jairouter.jar --generate-key

# Set environment variables
export JWT_SECRET="generated-key"
export INITIAL_ADMIN_PASSWORD="strong-password"

# Start container
docker run -d \
  --name jairouter \
  -p 8080:8080 \
  -e JWT_SECRET="$JWT_SECRET" \
  -e INITIAL_ADMIN_PASSWORD="$INITIAL_ADMIN_PASSWORD" \
  -e GPUSTACK_API_TOKEN="your-api-token" \
  sodlinken/jairouter:latest
```

### Docker Compose

```yaml
version: '3.8'
services:
  jairouter:
    image: sodlinken/jairouter:latest
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - INITIAL_ADMIN_PASSWORD=${INITIAL_ADMIN_PASSWORD}
      - GPUSTACK_API_TOKEN=${GPUSTACK_API_TOKEN}
    ports:
      - "8080:8080"
```

### Upgrade Steps

1. Backup current configuration and data
2. Generate new keys (if needed)
3. Update environment variable configuration
4. Restart application
5. Verify security check results in startup logs

## ⚠️ Important Notes

### Security Warnings

If weak keys or passwords are detected on startup, the application will output warnings:

```
╔══════════════════════════════════════════════════════════════════════════════╗
║  ⚠️  Security Warning: Weak key or password detected                          ║
║                                                                              ║
║  Before using in production, please ensure:                                 ║
║  1. Set strong key: export JWT_SECRET="<random 32+ byte key>"               ║
║  2. Set strong password: export INITIAL_ADMIN_PASSWORD="<complex password>" ║
║  3. Use key generation tool: java -jar jairouter.jar --generate-key         ║
╚══════════════════════════════════════════════════════════════════════════════╝
```

### Backward Compatibility

- ✅ Configuration format is compatible, no need to modify existing config files
- ⚠️ Hardcoded keys need to be replaced with environment variable references
- ✅ Default values are still available, but security warnings will be issued
