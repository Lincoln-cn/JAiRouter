## 注意事项

1. **使用 rebase 而非 merge**: 本次合并使用 `git rebase` 而非 `git merge`，保持了提交历史的线性，避免了复杂的合并提交。

2. **冲突解决原则**: 优先使用 origin (Gitee) 的版本，因为它是上游仓库，版本管理更加规范。

3. **文档更新提交**: 大量的 `docs: 更新文档版本信息` 提交由 GitHub Actions 自动生成，这些提交在 rebase 过程中被自动跳过（重复）。

4. **版本号管理**: 建议在合并后统一版本号管理，避免不同仓库之间的版本冲突。
